# External Integrations

**Analysis Date:** 2026-07-08

## APIs & External Services

**Nostr Relays:**
- NIP-01 relay WebSocket network - reads, subscribes, publishes, counts, and negentropy-syncs events.
  - SDK/Client: `rxjs/webSocket`, `nostr-tools`, and Applesauce relay classes in `packages/relay/src/relay.ts`, `packages/relay/src/pool.ts`, and `packages/loaders/src`.
  - Auth: NIP-42 signed AUTH events using `nostr-tools/nip42` in `packages/relay/src/relay.ts`; caller supplies an `AuthSigner`.
- NIP-86 relay management HTTP JSON-RPC - manages relay allow/ban lists, metadata, event moderation, kind policies, and IP blocks.
  - SDK/Client: native `fetch` plus `nostr-tools/nip98` token generation in `packages/relay/src/management.ts`.
  - Auth: `Authorization: Nostr <token>` generated by signer in `packages/relay/src/management.ts`.
- NIP-11 relay information - relay metadata types are imported from `nostr-tools/nip11` in `packages/relay/src/types.ts`.
  - SDK/Client: relay package types and HTTP URL normalization in `packages/relay/src/types.ts` and `packages/relay/src/management.ts`.
  - Auth: Not required for public relay information.

**Cashu / Lightning:**
- Cashu mints and wallets - NIP-60 wallet operations mint, melt, receive, consolidate, and roll over tokens.
  - SDK/Client: `@cashu/cashu-ts` in `packages/wallet/package.json`, `packages/wallet/src/wallet/nut-wallet.ts`, and `packages/wallet/src/actions/*`.
  - Auth: Cashu proof/token cryptography; no repository-level env auth detected.
- BOLT11 invoice decoding - parses Lightning invoices for Nostr zap and wallet helpers.
  - SDK/Client: `light-bolt11-decoder` in `packages/common/package.json`.
  - Auth: Not applicable.

**Blossom / Media Upload:**
- Blossom file servers - reference app has Blossom upload helpers and Concord exposes an upload injection interface.
  - SDK/Client: `blossom-client-sdk` in `apps/examples/package.json` and `refs/accordian/src/lib/blossom.ts`; core Concord uses injected `ConcordUploader` in `packages/concord/src/storage.ts`.
  - Auth: signed Blossom upload auth through `createUploadAuth` in `refs/accordian/src/lib/blossom.ts`; no env variable detected.

**Documentation / Reference:**
- VitePress docs site - builds and serves package docs with local search and edit links.
  - SDK/Client: `vitepress` and `vue` in `apps/docs/package.json`; config in `apps/docs/.vitepress/config.ts`.
  - Auth: Not applicable.
- TypeDoc API reference - generates API docs from TypeScript packages.
  - SDK/Client: `typedoc` in `package.json` and `typedoc.json`.
  - Auth: Not applicable.

**Browser and Device APIs:**
- WebSocket - relay clients and local relay server communicate over WebSockets.
  - SDK/Client: `rxjs/webSocket` in `packages/relay/src/relay.ts`; `ws` in `packages/sqlite/src/relay.ts`.
  - Auth: NIP-42 for relay connections when required.
- Web Serial - hardware signer support is gated on `navigator.serial` in `packages/signers/src/signers/serial-port-signer.ts`.
  - SDK/Client: browser `navigator.serial` API.
  - Auth: User grants browser device permission.
- Android/Capacitor signing - optional Amber/native signer support.
  - SDK/Client: optional `nostr-signer-capacitor-plugin` in `packages/signers/package.json` and `packages/accounts/package.json`.
  - Auth: Mobile OS/plugin account authorization.

## Data Storage

**Databases:**
- In-memory `EventStore` - default event cache and reactive event source in `packages/core/src/event-store`.
  - Connection: constructed in code; no env vars.
  - Client: Applesauce `EventStore`.
- SQLite via `better-sqlite3` - synchronous local event database and backing store for local relay.
  - Connection: `DATABASE_PATH` in `packages/sqlite/src/relay.ts`; defaults to `:memory:`.
  - Client: `BetterSqlite3EventDatabase` in `packages/sqlite/src/better-sqlite3/event-database.ts`.
- Node native SQLite - Node `node:sqlite` adapter for environments with native SQLite support.
  - Connection: caller-provided database path/options in `packages/sqlite/src/native/event-database.ts`.
  - Client: `DatabaseSync` in `packages/sqlite/src/native/methods.ts`.
- libSQL - remote/local libSQL adapter.
  - Connection: caller-provided `@libsql/client` config in `packages/sqlite/src/libsql/event-database.ts`.
  - Client: `@libsql/client` `createClient`.
- Turso native and Turso WASM - embedded/browser-compatible database adapters.
  - Connection: caller-provided database configuration in `packages/sqlite/src/turso/event-database.ts` and `packages/sqlite/src/turso-wasm/event-database.ts`.
  - Client: `@tursodatabase/database` and `@tursodatabase/database-wasm`.
- Bun SQLite - Bun runtime adapter.
  - Connection: caller-provided database path/options in `packages/sqlite/src/bun/event-database.ts`.
  - Client: `bun:sqlite`.

**File Storage:**
- Browser/local filesystem indirectly through SQLite adapters - `packages/sqlite/src/relay.ts` uses `DATABASE_PATH` for a SQLite file when configured.
- Blossom-compatible file servers - upload implementation is injected through `ConcordUploader` in `packages/concord/src/storage.ts`; reference implementation imports `blossom-client-sdk` in `refs/accordian/src/lib/blossom.ts`.
- No cloud object storage SDKs such as S3/GCS/Azure detected.

**Caching:**
- RxJS `shareReplay` and in-memory maps - used throughout relay, wallet, and store flows, including mint/client caches in `packages/wallet/src/wallet/nut-wallet.ts`.
- Browser `localStorage` - wallet couch in `packages/wallet/src/helpers/local-storage-couch.ts`, Concord storage defaults in `packages/concord/src/storage.ts`, and reference app account/cache storage in `refs/accordian/src/nostr.ts` and `refs/accordian/src/concord/cache.ts`.
- Browser IndexedDB - wallet couch implementation in `packages/wallet/src/helpers/indexed-db-couch.ts`.
- Pluggable encrypted-content cache - optional `EncryptedContentCache` in `packages/wallet/src/wallet/nut-wallet.ts`.
- No Redis/Memcached integration detected.

## Authentication & Identity

**Auth Provider:**
- Custom Nostr identity and signing - package signers create and authorize events rather than using OAuth/SAML.
  - Implementation: `applesauce-signers` package in `packages/signers/package.json`; helpers use `nostr-tools` for key derivation, NIP-19, NIP-44, and NIP-49 in `packages/core/src/helpers/*`.
- Relay authentication - NIP-42 challenge/response using signed AUTH events.
  - Implementation: `makeAuthEvent` from `nostr-tools/nip42` in `packages/relay/src/relay.ts`.
- Relay management authentication - NIP-98 HTTP auth tokens.
  - Implementation: `getToken` from `nostr-tools/nip98` in `packages/relay/src/management.ts`.
- Nostr Wallet Connect - NWC client/service protocol helpers.
  - Implementation: `packages/wallet-connect/package.json`, `packages/wallet-connect/src/wallet-connect.ts`, and `packages/wallet-connect/src/wallet-service.ts`.
- Android/Amber signer integration - optional mobile signer plugin.
  - Implementation: optional `nostr-signer-capacitor-plugin` in `packages/signers/package.json` and `packages/accounts/package.json`; gated exports in `packages/signers/src/signers/index.ts` and `packages/accounts/src/accounts/index.ts`.
