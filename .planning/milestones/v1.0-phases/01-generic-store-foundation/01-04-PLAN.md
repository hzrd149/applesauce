<objective>
Genericize `EventStore` and `AsyncEventStore` over `E extends StoreEvent = NostrEvent` (CORE-01, CORE-02) and land the one intentional runtime change of this phase: the constructor honoring an explicit `verifyEvent: undefined` to disable verification (CORE-03), while keeping the setter's `console.warn` exactly as-is (D-01).

Purpose: This is the capstone of the Phase 1 migration. It ties together the generic helpers (Plan 01), interfaces (Plan 02), and managers (Plan 03) so `new EventStore()` remains a signed `NostrEvent` store while `EventStore<Rumor>` (used by Phase 3's `RumorStore`) becomes expressible. The `verifyEvent` option/property becomes generic over `E` (D-04). The subscription/model methods inherited from the non-generic `EventModels` stay `NostrEvent`-typed — the accepted D-02 type gap that Phase 2 closes.

Output: Two generic store classes, the CORE-03 constructor fix in both, a focused runtime test for the fix, and a green build + full test suite with zero behavior change for default signed stores.
</objective>
