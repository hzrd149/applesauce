<objective>
Genericize the four event-store managers — `DeleteManager`, `AsyncDeleteManager`, `ExpirationManager`, `EventMemory` — over `E extends StoreEvent = NostrEvent` (CORE-05, manager portion), preserving current runtime behavior exactly.

Purpose: The store classes (Plan 04) type their `database`, `memory`, `deletes`, and `expiration` fields with these managers. The managers must be generic — with `NostrEvent` defaults — so Plan 04 can thread `E` through the store. This plan depends on Plan 01 (generic helpers the managers call) and Plan 02 (generic interfaces the managers implement).

Output: Four generic manager classes with `NostrEvent` defaults, localized bridge casts where they call still-`NostrEvent`-typed helpers, and a green build + full test suite.
</objective>
