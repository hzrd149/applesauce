Deferred findings from the Phase 05.1 advisory code review (`05.1-REVIEW.md`). WR-02 was
fixed in-phase (commit 44ec397f); the items below were triaged as pre-existing or
documented known-gaps and backlogged per user decision. None is a Phase 05.1 regression.

## CR-01 (BLOCKER, pre-existing — 2025-11-25) — author-spoofing via discarded verify
`packages/common/src/helpers/gift-wrap.ts:209` — `getGiftWrapSeal` calls `verifyWrappedEvent(seal)`
but discards the boolean, so a seal with an invalid signature is accepted and cached; combined
with the `rumor.pubkey === seal.pubkey` binding against an unverified `seal.pubkey`, this is an
author-spoofing bypass. The concord sibling (`packages/concord/src/helpers/gift-wrap.ts`) guards
correctly: `if (!verifyWrappedEvent(seal)) return null;`. **Fix:** gate on the return value.
Security-relevant — prioritize despite being pre-existing.

## WR-01 (WARNING, pre-existing) — replaceable version symbol copy
`packages/core/src/event-store/event-store.ts` `copySymbolsToDuplicateEvent` — the CR-04 `||`
guard (fixed this phase) is correct, but the replaceable sameness check compares only
kind+pubkey+identifier, not `id`/content, so `verifiedSymbol` and `EncryptedContentSymbol` can
copy between *different versions* of the same replaceable event (wrong plaintext / unearned
verified flag). **Fix:** gate those two symbols on `source.id === dest.id`.

## WR-03 (WARNING, documented known-gap) — getSealRumor undefined sentinel
`getSealRumor`'s `undefined` sentinel + `isSealUnlocked`'s presence-only check lets `unlockSeal`
return `undefined` typed as `Rumor` (WR-10 in prior review register). Left untouched per plan scope.

## WR-04 (WARNING, latent/unused) — EventFactory.kind() never resolves
`packages/core/src/factories/event.ts` `kind()` — the executor param is named `resolve` but the
value is spread-and-discarded, so the returned factory promise never resolves (latent deadlock).
Currently unused. **Fix:** resolve the promise with the new-kind template.
