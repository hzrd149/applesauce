<objective>
Genericize the remaining cast infrastructure named in CORE-07 — `CastRefEventStore`, `CastConstructor`, `castEvent`, `castEventStream`, `castTimelineStream` — over `E extends StoreEvent = NostrEvent`, so casts compose over any store event. `EventCast<T>` itself is already generic (prior commit `82c8839c`); this plan supplies the missing other half. Then run the final full-workspace build gate for the whole phase.

Purpose: `CastRefEventStore` composes `EventModels`, so it could not be genericized until Plan 02 made `EventModels<E>` generic — this plan is strictly after Wave 2. The existing `rumor-cast.test.ts` already proves `EventCast<Rumor>` works structurally against a bare store; this plan makes the `store` type properly `E`-aware so a Phase 3 `RumorStore`-backed cast is typed correctly rather than compatible-by-accident. Zero runtime behavior changes — every parameter defaults to `NostrEvent`.

Output: Generic cast infrastructure with `NostrEvent` defaults, the documented contravariance trick preserved, a `minor` changeset, the two pre-existing cast regression tests passing unmodified, and a green FULL-workspace build (the phase gate).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw stored event → typed cast instance | `castEvent`/`CastRefEventStore` wrap a raw store event in a typed convenience class consumed in both Node and browser contexts. The boundary is compile-time typing over an already-stored event; no new untrusted input is parsed. The contravariance trick keeps the constructor `event` param narrow (`NostrEvent`) so a rumor cast cannot silently be constructed where a signed event is required. |
