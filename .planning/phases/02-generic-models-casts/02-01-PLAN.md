<objective>
Genericize the two leaf claim operators (`claimEvents`, `claimLatest`) and the four base models (`EventModel`, `ReplaceableModel`, `TimelineModel`, `FiltersModel`) over `E extends StoreEvent = NostrEvent`, so the reactive read layer produces `E`-typed observables. This is Wave 1 of CORE-06 — the leaf dependencies that Wave 2 (the `EventModels`/interface seam) builds on top of.

Purpose: `TimelineModel`/`EventModel`/`ReplaceableModel` call `claimEvents`/`claimLatest` directly, so those operators MUST be generic first or the model bodies will not type-check once they operate on `E` instead of `NostrEvent`. The base model functions are exported (via the `Models` namespace) so genericizing them is itself a public, minor change. This plan changes zero runtime behavior — every generic parameter defaults to `NostrEvent`, so `TimelineModel(...)` and every existing call site resolve exactly as before.

Output: Two generic claim operators, four generic base models with generic private helpers, one localized bridge cast for `insertEventIntoDescendingList`, a `minor` changeset, and a green core build + full test suite.
</objective>
