<objective>
Close the D-02 / WR-02 seam left open by Phase 1: thread `E` through `IEventSubscriptions<E>` (fill in the currently-dead type parameter), `Model`, `ModelConstructor`, `ModelEventStore`, and `EventModels`, then update `EventStore<E>`/`AsyncEventStore<E>` to extend `EventModels<E, ...>`. After this plan, `EventStore<E>.timeline()`/`event()`/`filters()` truly return `E`-typed observables instead of dropping `E` to `NostrEvent`.

Purpose: This is the highest-regression-risk change in the phase. `IEventSubscriptions<E>` already declares its type parameter (Phase 1 Plan 02) but every method body still hardcodes `NostrEvent` — resolving WR-02 means making that dead parameter live. Because `EventModels` gains a SECOND type parameter, the six downstream `declare module "applesauce-core/event-store" { interface EventModels { ... } }` augmentations (which declare zero type parameters) must be re-verified with a full-workspace build — this is RESEARCH Pitfall 2, the single untested risk of the phase. The full `pnpm -r build` in this plan is the mandated gate that must pass BEFORE Wave 3 (cast work) builds on `EventModels<E>`.

Output: A genericized interface + model-mixin layer, `EventStore<E>`/`AsyncEventStore<E>` extending `EventModels<E, ...>`, a `minor` changeset, and a green FULL-workspace build (all packages) plus the core test suite.
</objective>
