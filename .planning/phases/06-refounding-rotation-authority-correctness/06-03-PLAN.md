<objective>
Fix H03 / AUTH-01 + AUTH-02: a root Refounding's removal must be honored only from a rotator who **strictly outranks every removed target** — "in both" the send and receive paths (CORD-06 §3). Both halves already exist correctly for channels; the fix is to mirror the channel path onto the root path (RESEARCH: "mirror the channel path onto the root path").

- **Send (AUTH-02, D-05/D-06):** `refound()` (`community.ts:1055-1106`) only checks `refoundAuthority` (bare BAN bit, no rank). Add a per-target BAN outrank loop mirroring `rotateChannel` (`community.ts:885-888`, swap `MANAGE_CHANNELS`→`BAN`), throwing before anything is built/published — atomic abort.
- **Receive (AUTH-01, D-07/D-08):** `readRekeyScoped:506`'s `(!held.canRemoveSelf || held.canRemoveSelf(...))` defaults to PERMIT — the milestone's recurring "guard defaults to permit" defect. Make it fail-closed (`held.canRemoveSelf?.(set.rotator) === true`), add a `canRemoveSelf` parameter to the root `readRekey` (mirroring `readChannelRekey`), supply it from both `readRekey` call sites via `canActOn(...PERM.BAN)`, and rewrite the misleading `keys.ts:447-454` docstring that declares the omission intentional.

Purpose: close two Elevation-of-Privilege holes (any BAN holder can currently evict the owner on receive; `refound` never rank-checks on send) with the roster-based rank primitive the codebase already uses. Rank semantics are inherited, not reinvented (D-09).
Output: the send-path outrank loop, the fail-closed receive path with both call sites wired, their spec-derived tests, and two patch changesets.

## Coupling note (must land together)
D-07 (fail-closed) and D-08 (supply the predicate) are inseparable: once `readRekeyScoped` denies removal when `canRemoveSelf` is absent, the root `readRekey` MUST supply it at BOTH call sites or legitimate Refounding-removal silently breaks. Task 2 changes the fail-closed branch, the `readRekey` signature, and both callers in one unit. Existing `readRekey` tests that assert `removed` without a predicate must be updated in the same task.
</objective>
