  const keys = deriveChannelKeys(ctx.material, channel);

Instrumented (aggregate uses `channel.epoch` — the enclosing loop's known epoch value for this file, per Pitfall 3 — not `info.epoch`):
```typescript
async function syncMessagePlanes(ctx: ChannelSyncContext, channel: ChannelKey): Promise<void> {
  const keys = deriveChannelKeys(ctx.material, channel);
  const streamKeys = [keys.current, ...keys.held.map((h) => h.key)];
  ctx.relayAuth.registerStreamKeys(streamKeys);
  ctx.ensureAuth(ctx.relays);
  const fetched = await syncAuthors(ctx, streamKeys.map((k) => k.pk));
  let decoded = 0, dropped = 0;
  for (const ev of fetched) {
    const info = keys.planes.get(ev.pubkey);
    if (!info || info.type !== "channel") continue;
    const d = decodeWrapCached(ev, info.convKey);
    if (!d) {
      dropped++;
      ctx.logger.extend("decode")("dropped wrap %s plane=channel epoch=%d", ev.id.slice(0, 8), channel.epoch);
      continue;
    }
    decoded++;
    ctx.route(info, d);
  }
  ctx.logger("message planes: fetched=%d decoded=%d dropped=%d epoch=%d", fetched.length, decoded, dropped, channel.epoch);
}
```
Same pattern for `syncRekeyAndAdvance` (`channel-sync.ts:66-98`), aggregating over `keys.nextRekey` wraps.

## Shared Patterns

### Base logger export (reused as-is, no changes)
**Source:** `packages/core/src/logger.ts` (full file, verified current)
```typescript
import debug from "debug";
export const logger = debug("applesauce");
```
**Apply to:** every concord file as the fallback base — `import { logger } from "applesauce-core";` then `logger.extend("concord")`.

### Injectable logger with module fallback (the wallet template)
**Source:** `packages/wallet/src/wallet/nut-wallet.ts:245`, `packages/wallet/src/wallet/types.ts:47-48`
```typescript
logger?: Debugger; // options field
this.log = options.logger ?? baseLogger.extend("nut-wallet").extend(this.pubkey.slice(0, 8)); // constructor
```
**Apply to:** `ConcordClientOptions`, `ConcordCommunityOptions`, `ConcordPrivateChannelOptions`, `ConcordInviteManagerOptions`, `InviteWatcherOptions`.

### `.extend()` per-instance/per-area precedent
**Source:** `packages/core/src/event-store/event-memory.ts:16` (`protected log = logger.extend("EventMemory");`), `packages/core/src/helpers/event-cache.ts:7` (`const log = logger.extend("event-cache");`)
**Apply to:** every module-level or class-level logger derivation across the sweep.

### Decode-boundary aggregate + per-wrap logging (the crux, D-05/D-06/D-07)
**Source:** synthesized from `packages/concord/src/client/sync.ts:136-169` (existing loop shape) — see full pattern under `sync.ts` above.
**Apply to:** `sync.ts` `syncEpoch` steps 2 & 3, `channel-sync.ts` `syncMessagePlanes` & `syncRekeyAndAdvance`, and the live `onWrap` sites in `community.ts`/`private-channel.ts` (single-event variant, no aggregate).
**Critical gotcha:** always source the epoch from the enclosing loop's known value (`epochMaterial.root_epoch` in `sync.ts`, `channel.epoch` in `channel-sync.ts`), never `info.epoch` (undefined for control/dissolved/rekey plane types — `keys.ts:48-53`).

### D-09 dual-emit for genuine failures
**Source:** synthesized from `community.ts:1061` shape — see `invite-manager.ts` section above.
**Apply to:** all 15 console.warn/console.error sites listed in CONTEXT.md's canonical refs across `relay-auth.ts`, `invite-manager.ts`, `invite-watcher.ts`, `client.ts`, `community.ts`, `private-channel.ts`.
