## Logger mechanism & injectability

| Option | Description | Selected |
|--------|-------------|----------|
| core logger + extend | Reuse applesauce-core's `logger`, `.extend("concord")`, opt-in via DEBUG, no new dep | |
| + injectable override | Same core-logger base PLUS optional `logger?: Debugger` on client/community/channel options (mirrors wallet) | ✓ |
| console.debug only | Plain `console.debug`, no `debug` package | |

**User's choice:** + injectable override
**Notes:** Matches the applesauce-wallet nut-wallet pattern; `applesauce-core` already a dep so no new package.

---

## Breadth — "across the board"

| Option | Description | Selected |
|--------|-------------|----------|
| Sync-focused (additive) | Add logging to sync/decode paths only, leave console.* untouched | |
| Sync + convert existing | Add sync logging AND route existing console.* through the namespaced logger | |
| Full sweep | Sync logging plus logging at every notable operation (auth, publish, invite lifecycle, folds), converting console.* usage | ✓ |

**User's choice:** Full sweep
**Notes:** Sync/decode remains the deepest/most detailed; the rest gets lighter operational tracing.

---

## Decode-boundary granularity

| Option | Description | Selected |
|--------|-------------|----------|
| Aggregate counts | Per plane/epoch fetched/decoded/dropped counts | |
| Counts + per-wrap | Aggregate counts PLUS a per-failed-wrap line (wrap id, plane, epoch) | ✓ |
| Counts + failure reason | Aggregate counts + crack decodeWrap's null into distinct reasons | |

**User's choice:** Counts + per-wrap
**Notes:** Pinpointing individual failures chosen over cracking the collapsed `null`; `decodeWrap`/`decodeWrapCached` left unmodified. Reason-cracking noted as a deferred idea.

---

## Log visibility level

| Option | Description | Selected |
|--------|-------------|----------|
| Opt-in debug | Silent unless DEBUG=applesauce:concord* set; consistent with core | ✓ |
| Warn by default | Decrypt-failure surfaces at console.warn (always visible) | |

**User's choice:** Opt-in debug
**Notes:** "Observable" = available when you look. Applies to the diagnostic/sync-decode path.

---

## Error-path handling (raised by Full sweep + Opt-in debug tension)

| Option | Description | Selected |
|--------|-------------|----------|
| Keep errors visible | Diagnostics → debug namespace; genuine failures stay console.warn/error | |
| All to debug | Everything (incl. console.warn/error) through the namespaced logger; real failures become silent unless DEBUG set | |
| Both for errors | Diagnostics → debug namespace; genuine failures fire BOTH the debug logger AND console.warn/error | ✓ |

**User's choice:** Both for errors
**Notes:** `debug` package has no level tiers, so routing errors through it alone would silence them; dual-emit keeps them visible and also captured under an injected logger.

---

## Per-wrap verbosity namespacing

| Option | Description | Selected |
|--------|-------------|----------|
| Split namespaces | Summary counts at `…:sync`; per-failed-wrap lines at a finer child (`…:sync:decode`) enabled independently | ✓ |
| Single namespace | Counts and per-wrap both under one sync namespace | |

**User's choice:** Split namespaces
**Notes:** Keeps summary readable; per-wrap detail is on-demand rather than flooding whenever sync logging is enabled.
