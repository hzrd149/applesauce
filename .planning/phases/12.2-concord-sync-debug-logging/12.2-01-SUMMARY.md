# Phase 12.2 Plan 01: Concord Client Logging Foundation Summary

Threaded an injectable, namespaced `debug`-based logger through Concord's `ConcordClient` -> `ConcordCommunity` -> `ConcordPrivateChannel` hierarchy plus the two invite classes, and gave `SyncContext`/`ChannelSyncContext` a required `logger` handle — the plumbing every later plan's actual log emission depends on.

## What Was Built

**Task 1 — Base logger module + dependency correction.** Added `debug`/`@types/debug` as concord's own direct `dependencies`/`devDependencies` (RESEARCH found CONTEXT.md's D-01 "transitive" assumption was empirically false — `tsc` fails with `TS2307` without this). Created `packages/concord/src/logger.ts` exporting `logger = coreLogger.extend("concord")`, the `applesauce:concord` namespace root.

**Task 2 — Injectable logger threading.** Added `logger?: Debugger` to `ConcordClientOptions`, `ConcordCommunityOptions`, `ConcordPrivateChannelOptions`, `ConcordInviteManagerOptions`, `InviteWatcherOptions`. Each class derives `this.log` in its constructor via the `options.logger ?? <base>` fallback idiom (mirroring `applesauce-wallet`'s `nut-wallet.ts`):
- `ConcordClient.log = options.logger ?? logger` (no per-instance suffix — one client per session)
- `ConcordCommunity`/`ConcordPrivateChannel.log = options.logger ?? logger` (base; the real per-instance namespace arrives via the parent's construction-site extend)
- `ConcordInviteManager`/`InviteWatcher.log = options.logger ?? logger.extend("invite")`

Threaded at each construction site: `ConcordClient.addCommunity()` passes `this.log.extend("community").extend(material.community_id.slice(0,8))`; `ConcordCommunity.spawnPrivateChannel()` passes `this.log.extend("channel").extend(channelKey.id.slice(0,8))`; the client's `ConcordInviteManager`/`InviteWatcher` constructions both pass `this.log.extend("invite")`.

**Task 3 — Required SyncContext.logger.** Added a required `logger: Debugger` field to `SyncContext` (`ChannelSyncContext` inherits it via `extends SyncContext`). `community.ts`/`private-channel.ts`'s `syncContext()` methods satisfy it with `logger: this.log.extend("sync")` — producing the `applesauce:concord:community:<id8>:sync` / `applesauce:concord:channel:<id8>:sync` namespaces plan 02's aggregate/per-wrap logging will emit under. Updated all 5 existing `SyncContext`/`ChannelSyncContext` test literals across `sync.test.ts`/`channel-sync.test.ts` with `logger: logger.extend("test")`.
