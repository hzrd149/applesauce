# Phase 05.1 Plan 05: Groups/App-Data Common Bug Fixes Summary

**Fixed two independent common-package defects (CR-03's leaky `lockAppData` and D-02/D-03's undefined-memoizing `getHiddenGroups`), each landed in its own commit with a hand-derived regression test that failed pre-fix and passes post-fix.**

## Performance

- **Duration:** ~15 min
- **Completed:** 2026-07-16
- **Tasks:** 2/2 completed
- **Files modified:** 4 (1 new test file, 1 modified source + modified test file per task)

## Accomplishments

- CR-03: `lockAppData` now performs a two-delete lock (deletes `AppDataContentSymbol` before calling `lockHiddenContent`), mirroring core's `lockHiddenTags` pattern, so a locked NIP-78 app-data event no longer leaks decrypted plaintext through `getAppDataContent`'s cached-read early return.
- `app-data.ts` gained its first behavior test file (`app-data.test.ts`), closing a Wave-0 zero-coverage gap.
- D-02/D-03: `getHiddenGroups` now returns before ever invoking `getOrComputeCachedValue` when the hidden tags are locked, guaranteeing the memo is only ever written when the computed value is a real `GroupPointer[]` ŌĆö never `undefined`. This closes the poisoning path where `isHiddenGroupsUnlocked`'s presence check was satisfied by a stale `undefined` memo and `unlockHiddenGroups`'s short-circuit handed that `undefined` back directly, bypassing its own `if (!groups) throw` guard.
- Deleted the now-fully-redundant `Reflect.set(bookmark, GroupsHiddenSymbol, groups)` at the old `groups.ts:139` and its ~20-line explanatory/deferral comment block ŌĆö not converted to `setCachedValue`, per the plan's explicit instruction (this is the one inventory site in the broader symbol-propagation migration that is deleted rather than migrated).
- `isHiddenGroupsUnlocked` and `unlockHiddenGroups` needed no changes ŌĆö both were already correct once the underlying memo could no longer be poisoned.

- Left `app-data.ts:71`'s enumerable `Reflect.set` write for `AppDataContentSymbol` unmigrated ŌĆö that's Plan 09's job (the `setCachedValue` non-enumerable migration), not this plan's.
