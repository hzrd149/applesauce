# Phase 05.1 Plan 02: CR-01/CR-04 Core Blocker Fixes Summary

**Two Phase-5-review blocker bugs fixed in isolation before their sites migrate: `isHiddenContactsUnlocked` no longer asserts a symbol it never checked (CR-01), and `copySymbolsToDuplicateEvent`'s replaceable guard fails closed on either-field mismatch instead of failing open (CR-04).**

## Performance

- **Duration:** ~10 min
- **Completed:** 2026-07-16
- **Tasks:** 2/2
- **Files modified:** 6 (2 source, 2 test, 2 changesets)

## Accomplishments

- Fixed `isHiddenContactsUnlocked` (`packages/core/src/helpers/contacts.ts:72-76`) to match the correct `bookmark.ts` pattern: `isHiddenTagsUnlocked(event) && (HiddenContactsSymbol in event || getHiddenContacts(event) !== undefined)`, closing the gap where `unlockHiddenContacts` could resolve `undefined` typed as `ProfilePointer[]`.
- Fixed `copySymbolsToDuplicateEvent`'s replaceable guard (`packages/core/src/event-store/event-store.ts:200-203`) so the pubkey/identifier mismatch check uses `||` instead of `&&`, meaning it now throws when **either** field differs rather than only when **both** differ — closing a data-exposure path where `verifiedSymbol` and decrypted `EncryptedContentSymbol` could merge onto a structurally unrelated replaceable/addressable event.
- Both regression tests were confirmed RED against the pre-fix source (captured in this session's tool transcript) before the fix landed, then confirmed GREEN after — each attributing to the bug, not to the future enumerable-to-non-enumerable migration.
- Neither fix touched the `Reflect.set` write sites slated for later migration: `contacts.ts:99` (deferred to Plan 07) and `event-store.ts:222`/`298` (deferred to Plan 08) are byte-for-byte unchanged — verified via `git diff 7da1c280..HEAD` limited to these two files.

## Decisions Made

- **Kind 3 (Contacts) registration scoped to test file only:** `canHaveHiddenTags(3)` was `false` everywhere in the codebase prior to this plan — no package registers `kinds.Contacts` for hidden-tags support via `setHiddenTagsEncryptionMethod`. Without registering it, the CR-01 regression test could never reach the guard-lie code path (`unlockHiddenTags` would throw "Event kind does not support hidden tags" immediately). Registered `setHiddenTagsEncryptionMethod(kinds.Contacts, "nip04")` at the top of `contacts.test.ts` only — this is test-fixture setup identical to what a downstream consumer app would do to enable NIP-02 private follows, not a change to library defaults, and stays within this plan's file scope.
- **Folded a prettier reformat into the second commit:** running `prettier --write` after Task 1's commit reformatted `contacts.test.ts`'s import wrap (whitespace-only). Rather than amend Task 1's commit (prohibited), the format fix was folded into Task 2's commit as a documented style cleanup on an already-touched file.
