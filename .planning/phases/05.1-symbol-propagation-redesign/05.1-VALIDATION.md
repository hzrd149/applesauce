# Phase 05.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Derived from `05.1-RESEARCH.md` § Validation Architecture. Each behavioral fix
> (CR-01..05 + `groups.ts`) is proven by its own spec-derived regression test
> landed BEFORE its site's migration; expected values are computed by hand from
> the spec, never read from the implementation under test (05-CONTEXT.md D-18).

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | vitest (pnpm monorepo workspace, `vitest.workspace.ts`) |
| **Config file** | `vitest.workspace.ts` (per-package configs under each `packages/*`) |
| **Quick run command** | `pnpm --filter <package> test` (e.g. `pnpm --filter applesauce-common test`) |
| **Full suite command** | `pnpm -r test` |
| **Estimated runtime** | ~TBD seconds (fill during Wave 0) |

## Sampling Rate

- **After every task commit:** Run the touched package's `pnpm --filter <package> test`
- **After every plan wave:** Run `pnpm -r test`
- **Before `/gsd-verify-work`:** Full suite must be green
- **Max feedback latency:** TBD seconds (fill during Wave 0)
