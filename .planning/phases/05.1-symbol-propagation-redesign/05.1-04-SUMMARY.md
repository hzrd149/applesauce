# Phase 05.1 Plan 04: CR-02 Unlock-Guard Family Summary

**Closed the CR-02 guard-lie in `emoji-pack.ts`/`mute.ts`/`trusted-assertions.ts` — each guard now derives every value its type predicate asserts (and caches it) instead of trusting a single stale symbol from an earlier partial read, proven by five hand-derived regression tests including a new trusted-assertions.test.ts file.**

## Accomplishments

- `isHiddenFavoriteEmojiPacksUnlocked` (the reachable defect) now requires `isHiddenTagsUnlocked(list) && getHiddenFavoriteEmojis(list) !== undefined && getHiddenFavoriteEmojiPackPointers(list) !== undefined` — the old any-of-four OR let a single symbol from a prior partial `getHiddenFavoriteEmojis` call short-circuit the guard to `true` while `FavoriteEmojiPacksHiddenPointersSymbol` was never derived, so `unlockHiddenFavoriteEmojiPacks` returned `packPointers: undefined` typed as `AddressPointer[]`
- `isHiddenMutesUnlocked` and `isHiddenProvidersUnlocked` now mirror the `bookmark.ts:82-86` two-clause shape (`isHiddenTagsUnlocked(x) && (Symbol in x || getDerivedValue(x) !== undefined)`) instead of `Symbol in x || isHiddenTagsUnlocked(x)` — the old shape let unlocked hidden *tags* alone (a lower-level symbol) report the event as fully unlocked without ever deriving the actual muted-things/providers value
- Created `trusted-assertions.test.ts`, closing the Wave 0 gap where this file had only exports-snapshot coverage and no behavior tests
- Five hand-derived regression tests (1 in emoji.test.ts, 2 in mutes.test.ts, 2 in trusted-assertions.test.ts) each confirmed RED against the pre-fix source and GREEN after the fix, by temporarily reverting the guard change via `git stash` and re-running the exact same test
- One changeset (`hidden-unlock-guard-family.md`, patch `applesauce-common`) covers all three fixes as one conceptual CR-02 fix, per the plan's explicit instruction

## Issues Encountered

Ran `pnpm --filter applesauce-core build` once to unblock local test runs — this is a pre-existing build-order dependency of the monorepo, not a code change, and nothing was committed for it.
- The first draft of the emoji-pack regression test asserted `isHiddenFavoriteEmojiPacksUnlocked(list)` returns `false` before calling `unlockHiddenFavoriteEmojiPacks`. This assertion doesn't hold post-fix: because the fixed guard *calls* `getHiddenFavoriteEmojiPackPointers(list)` as part of evaluating the AND chain, invoking the guard itself derives and caches the missing symbol, so the guard immediately reports `true` even in the "partial" state. Rewrote the test to assert on `Reflect.has(list, FavoriteEmojiPacksHiddenPointersSymbol)` (symbol presence) and on `unlockHiddenFavoriteEmojiPacks`'s returned `packPointers` value instead of the guard's boolean return — this is the actually-observable difference between the pre-fix and post-fix guard.
