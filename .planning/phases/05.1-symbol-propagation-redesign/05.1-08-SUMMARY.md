- `copySymbolsToDuplicateEvent`'s merge loop (`event-store.ts:222`) now writes `FromCacheSymbol`/`verifiedSymbol`/`EncryptedContentSymbol` via `setCachedValue`, non-enumerably
- `EventStoreSymbol` store-link write in sync `EventStore.add` (`event-store.ts:298`) migrated to `setCachedValue`

## Decisions Made
- Left the `!(symbol in dest)` merge gate and the CR-04-corrected replaceable guard (`event-store.ts:200-205`) completely untouched — this plan only touches the three writes, per the plan's own must-haves
- Excluded `verifiedSymbol` from the new merge-loop non-enumerability assertion: `FakeUser.event()` (test fixture) calls nostr-tools' `finalizeEvent`, which already sets `verifiedSymbol` on freshly built events. Because `dest` is itself built via `FakeUser.event()`, `verifiedSymbol` is already present on `dest` before `copySymbolsToDuplicateEvent` runs, so the `symbol in dest` gate would skip the merge for that symbol — asserting on it would test nostr-tools' write site, not this plan's. `EncryptedContentSymbol` (not set by `finalizeEvent`) exercises the same merge path cleanly.
- No changeset added, per the plan's explicit instruction — the mechanism changeset (Plan 01) governs this migration
