The enabling mechanism itself (unchanged, `packages/core/src/helpers/cache.ts:51-60`):
```ts
export function setCachedValue<T extends unknown>(event: any, symbol: symbol, value: T) {
  Object.defineProperty(event, symbol, { value, enumerable: false, writable: true, configurable: true });
}
export function getOrComputeCachedValue<T extends unknown>(event: any, symbol: symbol, compute: () => T): T {
  if (Reflect.has(event, symbol)) return Reflect.get(event, symbol);
  const value = compute();
  Object.defineProperty(event, symbol, { value, enumerable: false, writable: true, configurable: true });
  return value;
}
```

### Group C: Carry-forward addition to the two strip loops (mechanism sites, land FIRST)

**`packages/core/src/helpers/pipeline.ts:51-70`** (`pipeFromAsyncArray`), confirmed current shape:
```ts
export function pipeFromAsyncArray<T, R>(fns: Array<Operation<T, R>>, preserve?: Set<symbol>): Operation<T, R> {
  if (fns.length === 0) return identity as Operation<any, any>;
  return async function piped(input: T): Promise<R> {
    return fns.reduce(async (prev: any, fn: Operation<T, R>) => {
      const result = await fn(await prev);
      if (preserve && typeof result === "object" && result !== null && typeof prev === "object" && prev !== null) {
        const keys = Reflect.ownKeys(result).filter((key) => typeof key === "symbol");
        for (const symbol of keys) {
          if (!preserve.has(symbol)) Reflect.deleteProperty(result, symbol);
        }
      }
      return result;
    }, input as any);
  };
}
```

**Target addition** (carry-forward half, permanent; delete-loop half removed only after all Group A/B migrations land — RESEARCH.md Assumption A1):
```ts
return fns.reduce(async (prev: any, fn: Operation<T, R>) => {
  const prevValue = await prev; // hoist: must reuse the SAME resolved value for fn() and carry-forward (Pitfall 2)
  const result = await fn(prevValue);
  if (preserve && typeof result === "object" && result !== null && typeof prevValue === "object" && prevValue !== null) {
    const keys = Reflect.ownKeys(result).filter((key) => typeof key === "symbol");
    for (const symbol of keys) {
      if (!preserve.has(symbol)) Reflect.deleteProperty(result, symbol);
    }
    for (const symbol of preserve) {
      if (Reflect.has(prevValue, symbol) && !Reflect.has(result, symbol)) {
        Reflect.set(result, symbol, Reflect.get(prevValue, symbol));
      }
    }
  }
  return result;
}, input as any);
```

### Group D: `cache.ts` doc rewrite (D-05)

Replaces the ~115-line three-category taxonomy (`packages/core/src/helpers/cache.ts:1-60`, confirmed present this session, already marked "Superseded" at the top) with the one-rule block drafted in RESEARCH.md:
```ts
/**
 * One rule: every symbol write on an event-like object is non-enumerable
 * (via setCachedValue/getOrComputeCachedValue below). Nothing survives an
 * object copy implicitly — spread only copies enumerable own properties.
 * Carry-forward across factory-pipe operations is performed EXPLICITLY by
 * the pipe (pipeFromAsyncArray / EventFactory.chain), which copies every
 * symbol in PRESERVE_EVENT_SYMBOLS from each step's input to its output
 * after the operation runs. stamp()/sign() additionally re-copy
 * EncryptedContentSymbol via Reflect.has/get/set so a standalone
 * sign(signer)(draft) call (outside any pipe) still preserves plaintext.
 */
```

## Shared Patterns

### `getOrComputeCachedValue`/`setCachedValue` import
**Source:** `packages/core/src/helpers/cache.ts`, re-exported from `applesauce-core/helpers` and `applesauce-core/helpers/cache`.
**Apply to:** every Group A/B site. Most files already `import { getOrComputeCachedValue } from "applesauce-core/helpers"` (or a relative core import inside `applesauce-core` itself) — add `setCachedValue` as a second named import from the same module/specifier already in use in that file, rather than introducing a new import path.

### `PRESERVE_EVENT_SYMBOLS` allowlist
**Source:** `packages/core/src/helpers/pipeline.ts:5` (static `Set`).
**Apply to:** Group C (carry-forward loops in `pipeline.ts` and `factories/event.ts`), and any future symbol needing pipe survival — gift-wrap symbols become members at definition once moved to core, killing `common/operations/gift-wrap.ts:24-26`'s import-time mutation.

### Two-delete lock pattern
**Source:** `hidden-tags.ts`'s `lockHiddenTags` (delete raw + derived symbol together).
**Apply to:** CR-03 (`app-data.ts`'s `lockAppData`) and any future "lock" function — a lock must delete every symbol a corresponding getter reads, not just the innermost one.

### Copy-then-delete pattern
**Source:** `operations/event.ts`'s `stripSignature`/`stripStamp`.
**Apply to:** CR-05 (`stamp`) — never mutate the caller's `draft` object; always build the copy first, then delete from the copy.
