# Phase 05.1 Plan 11: Downstream setCachedValue migration (concord/wallet-connect/wallet) Summary

**Migrated all 9 remaining identity-memo writes across `applesauce-concord`, `applesauce-wallet-connect`, and `applesauce-wallet` onto `setCachedValue`, and closed the six Wave 0 test-coverage gaps those sites had.**

## Performance

- **Duration:** ~20 min
- **Tasks:** 3
- **Files modified:** 8 (source) + 6 new test files + 1 extended test file = 15 total touched

## Accomplishments

- concord's `decodeWrapCached` (gift-wrap.ts:119) now writes its memoized `DecodedEvent` bundle via `setCachedValue`, preserving the `Reflect.get` read and the `cached !== undefined` null-failure-sentinel that lets an undecryptable wrap short-circuit repeated decode attempts.
- wallet-connect's `notification.ts:92`, `request.ts:75`, and `response.ts:75` all write their parsed NIP-47 message memos via `setCachedValue`.
- wallet's `wallet.ts:62/82/103` (mints, private key, relays), `history.ts:94`, and `tokens.ts:92` all write via `setCachedValue`. The private-key write is a concrete security improvement: a plain spread of an unlocked wallet event no longer carries the decrypted `Uint8Array` private key along with it (T-05.1-05).
- Six new behavior test files close every Wave 0 gap this plan owned (concord gift-wrap; wallet-connect notification/request/response; wallet wallet/history), and `tokens.test.ts` gained a new non-enumerability case for `TokenContentSymbol`.

## Decisions Made

- Tested `decodeWrapCached`'s short-circuit behaviorally rather than via spies: decode a real wrap under the wrong `convKey` first (caches `null`), then retry with the now-correct key and assert the result is still `null` — proof the second call didn't re-attempt, without depending on whether `nip44`'s namespace export is spy-friendly.
- Built all test fixtures with each package's existing `EventFactory` subclasses (`WalletNotificationFactory`, `WalletRequestFactory`, `WalletResponseFactory`, `WalletFactory`, `WalletHistoryFactory`, `WalletTokenFactory`) rather than hand-rolled event literals — signing already carries the plaintext forward via `EncryptedContentSymbol`/hidden-tags carry-forward, so the memo write sites under test are exercised without an extra async decrypt round-trip.

None - plan executed exactly as written. All 9 write sites, all 6 new test files, and the tokens.test.ts extension match the plan's task list and acceptance criteria one-for-one.

## Issues Encountered

- The wallet package's `relaySet`/`normalizeURL` helper normalizes relay URLs to include a trailing slash; an early draft of `wallet.test.ts` asserted the un-normalized input string and failed. Corrected the test's expected value to the normalized form (`wss://relay.example/`) — not a source bug, a test-fixture correction caught immediately by running the suite.
- `isHistoryContentUnlocked` (an existing, unmigrated helper in `history.ts`) computes and memoizes `HistoryContentSymbol` as a side effect when the symbol isn't already set. An early draft of `history.test.ts` called it before asserting the "unattempted" (`HistoryContentSymbol in event === false`) precondition, which the side effect then falsified. Reordered the assertions so the unattempted check runs first — again a test-ordering fix, not a production change.
- Both packages' cross-package imports (`applesauce-core/helpers/*`) resolve through compiled `dist/` output via package.json `exports`, which was missing in this fresh worktree. Ran `pnpm turbo build --filter='./packages/*'` once at the start of execution to build all workspace packages before running any test; this is an environment precondition, not a plan deviation.
