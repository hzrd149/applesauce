# Phase 05.1 Plan 01: Move gift-wrap symbols to core; static PRESERVE_EVENT_SYMBOLS; carry-forward mechanism Summary

**GiftWrapSymbol/SealSymbol/RumorSymbol are now core-owned with a static, load-order-independent PRESERVE_EVENT_SYMBOLS, and both factory-pipe strip loops (pipeFromAsyncArray, EventFactory.chain) explicitly carry preserved symbols forward across each step's own internal spread.**

## Performance

- **Duration:** 6 min
- **Started:** 2026-07-16T14:35:15Z
- **Completed:** 2026-07-16T14:41:25Z
- **Tasks:** 2
- **Files modified:** 8 (1 new core module + 2 changesets + 5 modified files, plus 1 stale-snapshot fix)

## Accomplishments

- `packages/core/src/helpers/gift-wrap.ts` created, exporting `GiftWrapSymbol`/`SealSymbol`/`RumorSymbol` via the exact `Symbol.for("gift-wrap")`/`Symbol.for("seal")`/`Symbol.for("rumor")` registry keys previously defined in common — confirmed by direct ESM identity check that `core.SealSymbol === common.SealSymbol === Symbol.for("seal")`.
- `PRESERVE_EVENT_SYMBOLS` (`packages/core/src/helpers/pipeline.ts`) is now a static `Set` built with all four members (`EncryptedContentSymbol` + the three gift-wrap symbols) at construction time — no later mutation.
- `packages/common/src/operations/gift-wrap.ts`'s import-time `PRESERVE_EVENT_SYMBOLS.add(...)` calls (the WR-03 load-order-dependent widening) are deleted entirely.
- `packages/common/src/helpers/gift-wrap.ts` now re-exports the three symbols from `applesauce-core/helpers/gift-wrap` instead of defining its own `Symbol.for()` calls.
- `pipeFromAsyncArray` hoists `const prevValue = await prev` once and reuses it for both the operation call and a new carry-forward loop that restores any preserved symbol `prevValue` had that `result` is missing.
- `EventFactory.chain` gets the mirror carry-forward loop over `PRESERVE_EVENT_SYMBOLS` (`draft` → `result`).
- Both existing delete-loop halves are unchanged — only the carry-forward half was added, per scope (delete-loop removal is Plan 13's job).
