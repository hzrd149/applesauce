`05-VERIFICATION.md` offers two resolutions for CR-03: **delete** the false enforcement claim, or **rewrite** the test so the claim becomes true. **Take the rewrite branch.** Reasoning, stated here so the executor does not re-litigate it:

3. **D-15 locks the shape.** "The carry-forward half reproduces the audit's probe end-to-end — real factory pipe, real spread operations, real signing." The rewrite stays inside that shape; it adds one more real operation to the real pipe.

**Locked decisions:** D-13 (comment + two-sided test; the test enforces), D-14 (both halves in this one file — the contrast IS the lesson), D-15 (real pipe, real spreads, real signing), D-05 (write sites, not symbols), TEST-01 (non-vacuous; never assert against implementation output).

<planner_findings>
Verified empirically during planning (Node, against this working tree). The executor MUST re-confirm by reading the cited code, but these explain why the rewrite works and why the current test does not:

- **Why the current test cannot catch the migration.** After `modifyHiddenTags` returns, `sign` runs with no intervening spread. `stamp` and `sign` (`operations/event.ts`) copy `EncryptedContentSymbol` via `Reflect.has` → `Reflect.get` → `Reflect.set`, all of which are **enumerability-blind**. So a non-enumerable write at `modifyHiddenTags`'s return still reaches the signed event, and the suite stays green. `stamp`'s own `{ ...draft, pubkey }` spread is compensated by that explicit re-copy — which is exactly why it does not help.
- **Why `PRESERVE_EVENT_SYMBOLS` does not save it either.** `pipeFromAsyncArray`'s loop only DELETES non-listed symbols from each operation's result; it never copies a listed symbol from `prev` onto `result`. Membership is necessary, not sufficient.
- **What makes the rewrite work.** `modifyPublicTags` returns `{ ...draft, tags: … }` with **no symbol re-copy**. An enumerable symbol survives that spread; a non-enumerable one is dropped and nothing restores it. Confirmed in Node: `{...draft, tags:[…]}` drops a non-enumerable symbol and keeps an enumerable one. `includeAltTag` is a `modifyPublicTags` wrapper, so inserting it between `modifyHiddenTags` and `sign` places an uncompensated spread between the write and the read.
- **Net:** with the rewrite, migrating the write to a non-enumerable descriptor drops the plaintext at `modifyPublicTags` and both `getHiddenTags(signed)` and `getEncryptedContent(signed)` fail. The suite goes red. The claim becomes true.
</planner_findings>
