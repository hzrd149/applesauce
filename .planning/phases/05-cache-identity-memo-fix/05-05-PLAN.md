## Why these probes exist

A test only has value if it fails when the thing it guards breaks. This phase adds four assertions, and they split into two groups that fail for **different** reasons — a distinction that is easy to get backwards:

| New test | Turns RED when... | Probe |
|----------|-------------------|-------|
| `cache.test.ts` memo-drop half | cache memos become enumerable again | A |
| concord H01(a) control address | cache memos become enumerable again | A |
| concord H01(c) channel plane address | cache memos become enumerable again | A |
| `cache.test.ts` carry-forward half | a carry-forward symbol stops being preserved through the pipe | **B** |

**The carry-forward half must stay GREEN under Probe A.** It does not route through `cache.ts` — none of the plaintext write sites do. Expecting it to go red under Probe A is a misreading of the whole finding; if it DOES go red under Probe A, something is wrong with the fix's blast radius and that is a stop-and-report finding, not a pass.
