<objective>
Genericize the event-store interface set in `packages/core/src/event-store/interface.ts` over `E extends StoreEvent = NostrEvent` (CORE-05, interface portion), keeping the composite `IEventStore`/`IAsyncEventStore` interfaces implementable by the still-non-generic `EventModels` superclass.

Purpose: These interfaces are the contracts the managers (Plan 03) and the store classes (Plan 04) implement. They must be generic — with `NostrEvent` defaults — before those plans can thread `E` through. Because the subscription/model methods (`event`, `filters`, `timeline`, `model`, ...) are provided by the non-generic `EventModels` class that D-02 defers to Phase 2, the composite store interfaces must compose the subscription/model portions at their `NostrEvent` default so `EventStore<E> extends EventModels implements IEventStore<E>` still compiles in Phase 1.

Output: A fully generic interface file with `NostrEvent` defaults everywhere, a green build, and no changes required at any existing call site.
</objective>
