<objective>
Apply the transport-only extras to `ConcordCommunity` — the phase's primary site.
This file contains both halves of the phase: roughly a dozen transport call sites
that must union in the extras (D-12), and the protocol writes that must stay
provably clean (D-05), including one of the two dual-meaning locals where a single
local array feeds both a publish and a signed invite URL.

The structural guarantee is D-01: `relays()` is not widened. A new, differently
named private `transport()` performs the only merges, so a protocol write can only
pick up an extra if someone explicitly routed it through `transport()` — which the
prohibitions and the wave-5 canary both check.

Two correctness constraints must not move: the refounding majority arithmetic
(D-06 / ROTATE-09 — publish wider, count narrower), and `openLive`'s churn guard
(D-09 — a no-op extras emission must not reopen sockets).

Runs in wave 3 because it passes `extraRelays` into the private-channel option bag
declared by plan 02.

Purpose: the community engine is where most Concord traffic lives and where every
leak path enumerated in CONTEXT.md originates.
Output: `ConcordCommunityOptions.extraRelays` plus `transport()` and the reactive
derivations.

No changeset (D-15).
</objective>
