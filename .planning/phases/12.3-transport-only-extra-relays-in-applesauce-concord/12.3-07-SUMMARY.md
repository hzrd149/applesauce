# Phase 12.3 Plan 07: Reactivity/churn/auth/refounding-quorum tests for extras Summary

## Decisions Made
- The refounding quorum tests reuse the pre-existing D-09/D-11 majority-gate test's exact arithmetic shape (3 relays, threshold = ⌈(n+1)/2⌉ = 2) on a fixture distinct from that test's relay set, so the two tests' outcomes never share state
- "Extras cannot inflate" is proven by configuring extras to ALSO ack alongside the exact ack count that already passes without extras — a widened denominator would make this same count newly fall short
- "Extras cannot substitute" is proven by making protocol acks fall one short of threshold while the extras endpoint acks — misattributed acks would push the count over threshold
- The `connected$` reactivity test proves re-derivation by asserting the OLD extra's "up" status (pushed AFTER the extras value moved on) is ignored, before separately proving the NEW extra's "up" status alone flips `connected$` true — a stronger non-vacuity proof than checking only the new value's effect
- `invite-watcher`'s discovery-surface test asserts equality (not substring-absence) against the literal `INBOX_RELAYS` list, per the plan's explicit instruction, since equality also catches an accidentally dropped discovered inbox
- `private-channel`'s no-extras baseline test expects the trailing-slash-normalized form of its literal relay list, since `transport()`/`mergeRelaySets` has normalized both inputs unconditionally since plan 02 — this pins existing behavior, not a regression
- Every fixture relay hostname across all 13 new tests is distinct per test group so no assertion could pass by coincidental substring overlap
