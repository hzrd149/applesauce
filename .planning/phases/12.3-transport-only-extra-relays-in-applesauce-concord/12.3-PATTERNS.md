### `packages/concord/src/helpers/relays.ts` (NEW — utility, transform)

**Analogs:** `packages/core/src/observable/relay-selection.ts:51-61` (continuous `isObservable(x) ? x : of(x)` idiom) and `packages/core/src/helpers/relays.ts:47-72` (`mergeRelaySets`).

**Continuous observable-or-value idiom** (`relay-selection.ts:51-61`):
```typescript
export function ignoreBlacklistedRelays(
  blacklist: string[] | Observable<string[]>,
): MonoTypeOperatorFunction<ProfilePointer[]> {
  return pipe(
    combineLatestWith(isObservable(blacklist) ? blacklist : of(blacklist)),
    map(([users, blacklist]) => removeBlacklistedRelays(users, blacklist)),
  );
}
```
Applied directly (not as an operator, but as a plain `Observable<string[]>` factory) — see Anti-Pattern note below on why `unwrap` (`take(1)`) is wrong here.

**`mergeRelaySets`** (`packages/core/src/helpers/relays.ts:47-72`):
```typescript
/** Merge multiple sets of relays and remove duplicates (ignores invalid URLs) */
export function mergeRelaySets(...sources: (Iterable<string> | string | undefined | null)[]): string[] {
  const set = new Set<string>();
  for (const src of sources) {
    if (!src) continue;
    if (typeof src === "string") {
      try { set.add(normalizeRelayUrl(src)); } catch (error) { /* ignore */ }
    } else if (Reflect.has(src, Symbol.iterator)) {
      for (const url of src) {
        try { set.add(normalizeRelayUrl(url)); } catch (error) { /* ignore */ }
      }
    }
  }
  return Array.from(set);
}
```
Import as `import { mergeRelaySets } from "applesauce-core/helpers/relays";` (already a `dependencies` entry in `packages/concord/package.json`, not optional).

**New helper to write** (RESEARCH.md's recommended shape, composing the two analogs above):
```typescript
import { isObservable, Observable, of, startWith, distinctUntilChanged } from "rxjs";

function sameRelaySet(a: string[], b: string[]): boolean {
  if (a.length !== b.length) return false;
  const setB = new Set(b);
  return a.every((r) => setB.has(r));
}

/** Resolve an `extraRelays?: string[] | Observable<string[]>` option to a continuous
 *  Observable — NEVER take(1)s, so a long-lived engine reacts to every future update.
 *  For one-shot call sites, prefer applesauce-loaders' `unwrap` instead. */
export function toRelaysObservable(v: string[] | Observable<string[]> | undefined): Observable<string[]> {
  return (isObservable(v) ? v : of(v ?? [])).pipe(startWith([]), distinctUntilChanged(sameRelaySet));
}
```

## Shared Patterns

### Continuous-observable derivation (`extraRelays$`)
**Source:** `packages/core/src/observable/relay-selection.ts:51-61` (`isObservable(x) ? x : of(x)` idiom), composed into the new `toRelaysObservable()` helper in `packages/concord/src/helpers/relays.ts`.
**Apply to:** Every engine constructor that holds a long-lived socket or a status observable — `community.ts` (`connected$`/`authenticated$`, `openLive()`), `private-channel.ts` (same shape), `invite-watcher.ts` (its live sub).
**Do NOT use `unwrap` here** — `unwrap`'s hard-coded `take(1)` (`packages/loaders/src/helpers/loaders.ts:18-20`) freezes at the first emission; correct only for one-shot sites.

### One-shot snapshot merge (`unwrap` + `mergeRelaySets`)
**Source:** `packages/loaders/src/loaders/tag-value-loader.ts:72-90` (`unwrap(opts?.extraRelays, (extraRelays) => mergeRelaySets(pointer.relays, extraRelays))`).
**Apply to:** All per-operation call sites: publishes, `pool.request`, `syncAuthors`/`syncEpoch` backfill, invite-bundle fetch, `invite-manager.ts`'s entire surface (no live socket in that class).

### Relay-set merge primitive
**Source:** `packages/core/src/helpers/relays.ts:47-72` (`mergeRelaySets`).
**Apply to:** Every `transport()`/per-class merge helper (D-04) and every `unwrap` callback body above. Import path: `applesauce-core/helpers/relays` (already a `dependencies` entry, not optional).

### Per-class `transport()` merge helper (D-04)
**Source:** mirrors the existing `relays()` private accessor pattern at `community.ts:600-602` — same private-method shape, new name, calls `mergeRelaySets(this.relays(), <extras snapshot>)`. Applied uniformly across `ConcordCommunity`, `ConcordPrivateChannel`, `ConcordInviteManager`, `InviteWatcher` per RESEARCH.md's Open Question 2 recommendation (name collisions checked: `invite-watcher.ts` already has a public `relays$` for discovered inbox relays — a private `transport()` method coexists without collision).

### Debug logging for merged relay sets
**Source:** Phase 12.2's `applesauce:concord:*` namespaces (see `[[debug-logger-hoist-extend]]` — derive/store the `Debugger` once, never `.extend()` inline at a log call site). Apply this convention if adding trace logging at any new `transport()`/`toRelaysObservable` call site.
