<objective>
Create the one shared primitive every Concord engine will use to consume the new
transport-only `extraRelays` option: a continuous (never `take(1)`) resolver for
`string[] | Observable<string[]>`, a synchronous snapshot holder, and a merge
entry point that always routes through `mergeRelaySets` (D-02).

This plan is deliberately first and file-isolated: waves 2–4 all depend on the
`ExtraRelaysOption` type and the `ExtraRelays` holder existing, and the semantics
that D-10/D-11 hinge on (startWith empty, never freeze, churn-free re-emission)
are proven here once instead of four times.

Purpose: give every engine a single, tested way to answer "what are the extras
right now" without any of them re-deriving the `isObservable(x) ? x : of(x)`
idiom — and without any of them reaching for `unwrap`, whose hard-coded `take(1)`
would silently undo D-08/D-09 reactivity at the long-lived sites (D-11).
Output: `packages/concord/src/helpers/relays.ts`, exported from the helpers
barrel, plus its unit suite.

Per D-15 concord is unreleased: do NOT create a `.changeset/` file in this plan
or any other plan in this phase.
</objective>

  <action>
Write `packages/concord/src/helpers/relays.ts` exporting exactly four symbols and importing only from `rxjs` and `applesauce-core/helpers/relays`.

1. `export type ExtraRelaysOption = string[] | Observable<string[]>;` — JSDoc it as the transport-only extras option, explicitly stating it is never merged into a community's protocol relay set (D-01) and is purely additive (D-14).

2. `export function sameRelaySet(a: string[], b: string[]): boolean` — order-insensitive content equality: length check, then Set membership. Mirrors the existing `sameSet` comparator style in `packages/concord/src/client/community.ts`.

3. `export function toRelaysObservable(option?: ExtraRelaysOption): Observable<string[]>` — returns `isObservable(option) ? option : of(option ?? [])` piped through `startWith([] as string[])` then `distinctUntilChanged(sameRelaySet)`. JSDoc MUST state: this resolver is continuous by design and deliberately omits the first-value-only operator that applesauce-loaders' one-shot resolver applies, because Concord engines are long-lived and must react to every later emission (D-11); and that `startWith` means a not-yet-emitted source resolves to the empty set rather than blocking traffic (D-10).

4. `export class ExtraRelays` — the per-engine holder implementing D-04's "one place resolves the extras snapshot and dedupes":
   - constructor takes `option?: ExtraRelaysOption`; internally creates a `BehaviorSubject<string[]>` seeded `[]` and subscribes `toRelaysObservable(option)` into it, storing that `Subscription`. Subscribing in the constructor is what makes `current` correct synchronously for arrays and `BehaviorSubject` sources.
   - `readonly relays$: Observable<string[]>` — the subject as an Observable. Because the subject is the single subscriber of the caller's source, every consumer inside one engine shares one subscription. JSDoc MUST still recommend apps pass a hot/shared source (`BehaviorSubject`, or `shareReplay(1)`) because each engine constructs its own holder (D-10's documented consequence).
   - `get current(): string[]` — the subject's current value; the synchronous snapshot every per-operation call site reads (D-11's "per-operation snapshot" list).
   - `merge(base: string[]): string[]` — returns `mergeRelaySets(base, this.current)`. This is the ONLY place in the module that merges. JSDoc MUST state the result is a transport target set and must never be written into signed or published content.
   - `dispose(): void` — unsubscribes the source subscription and completes the subject.

Write `packages/concord/src/helpers/__tests__/relays.test.ts` covering every bullet in the behavior block above, using vitest and plain `BehaviorSubject`/`Subject` sources. Use two clearly distinct relay URLs plus one trailing-slash variant to prove normalization. Collect emissions into a local array via `.subscribe()` and assert on that array, so the "no further emission" cases are provable by length.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12.3-01 | Information Disclosure | `ExtraRelays.merge` | high | mitigate | `merge` takes an explicit `base` argument and returns a new array; it never mutates the base and never caches a merged protocol set. The helper has no reference to `JoinMaterial`, event templates, or storage — verified by the import-list acceptance criterion, so no protocol writer can reach it accidentally (D-01). |
| T-12.3-02 | Tampering | malformed / hostile relay URL in extras | medium | mitigate | All extras flow through `mergeRelaySets`, which normalizes via `normalizeRelayUrl` and silently drops entries that fail to parse — extras get identical validation to every other relay list in the monorepo (ASVS V5). A raw pass-through of `option` to any pool call is forbidden by design: `current`/`merge` are the only readers. |
| T-12.3-03 | Denial of Service | never-emitting extras Observable | medium | mitigate | `startWith([])` means a silent source resolves to the empty set immediately; no Concord operation can block waiting for the app's Observable (D-10). Asserted by the `toRelaysObservable(undefined)` and pending-Subject tests. |
| T-12.3-04 | Denial of Service | high-frequency extras emissions causing socket churn downstream | low | accept | `distinctUntilChanged(sameRelaySet)` collapses no-op re-emissions; genuine rapid changes are the app's own doing and debouncing is explicitly Claude's-discretion/deferred in CONTEXT.md. Consequence documented in JSDoc rather than mitigated. |
| T-12.3-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new packages are installed by this phase (RESEARCH.md "Package Legitimacy Audit: Not applicable"). `mergeRelaySets` comes from `applesauce-core` and the rxjs operators from `rxjs`, both already declared `dependencies` of `packages/concord`. If any task appears to require an install, stop and escalate. |
