<objective>
Build the phase's standing regression guard (D-05): one suite that constructs a
`ConcordClient` with a canary extras endpoint, drives the whole lifecycle through
the public API, and asserts the canary string appears in zero published event
payloads and zero returned artifacts — while positively asserting it DOES appear
as a transport target, so the suite proves the feature works rather than proving
the option is inert.

The canary half catches leak paths nobody enumerated, including ones later phases
introduce. The targeted half pins the four enumerated protocol writes individually
so a failure names its own cause instead of reporting "something leaked".

Per D-05 the critical framing: assertions target event CONTENT and returned
ARTIFACTS, never publish targets. A cache relay legitimately appearing as a
destination is the feature.

Runs in wave 5 alongside plan 07, which owns the reactivity/quorum tests in the
existing suites — no file overlap.

Purpose: make the never-persisted guarantee mechanically enforced rather than
argued.
Output: `packages/concord/src/client/__tests__/extra-relays.test.ts`.

No changeset (D-15).
</objective>
