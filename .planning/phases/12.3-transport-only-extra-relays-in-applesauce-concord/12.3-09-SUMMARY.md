# Phase 12.3 Plan 09: Refounding quorum single-relay-set derivation + invite-bundle relay hardening Summary

**`refound()` now computes its majority threshold and ack-attribution set from one deduplicated, fail-soft protocol relay set with tolerant ack-origin normalization, and `validateInviteBundle` filters `relays` entries to safe relay URL strings — closing the D-06/ROTATE-09 verification gap with 11 new regression tests.**

## Performance

- **Duration:** ~20 min
- **Tasks:** 2
- **Files modified:** 5 (2 source, 3 test)

## Accomplishments

- `refound()` (`packages/concord/src/client/community.ts`) derives a single local `protocolRelays` from `this.relays()` routed through `mergeRelaySets` — the same fail-soft normalize-and-deduplicate path every other relay list in the file already gets. Both `majorityThreshold` (`Math.ceil((protocolRelays.length + 1) / 2)`) and `protocolRelaySet` (`new Set(protocolRelays)`) are computed from that one value, so a trailing-slash duplicate or an unparseable entry can never make the denominator and the attribution set disagree, and can never throw a raw `Invalid URL` TypeError in place of the majority-abort rejection.
- Added a local `normalizeAckOrigin` helper inside `refound()` that returns `undefined` for an absent or unparseable publish-response `from`, swallowing the parse failure — mirroring the tolerance shape `relay-auth.ts`'s `lookupStatus` already establishes in this package. The ack filter now requires `ok === true` AND a defined normalized origin AND protocol-set membership; the intended majority-abort error is the only failure mode a caller can observe.
- The D-06 comment block above the quorum computation was rewritten to describe the fail-soft, bit-for-bit-when-well-formed guarantee the code now provides, rather than an unqualified bit-for-bit claim.
- `validateInviteBundle` (`packages/concord/src/helpers/invite-bundle.ts`) now filters the capped `relays` slice to entries that are `typeof === "string"` AND pass `isSafeRelayURL` (imported from `applesauce-core/helpers/relays`). The cap slice still runs first (unchanged allocation bound); entries are dropped, never rewritten (no normalization, per D-01); an entirely-junk relays array still validates successfully and falls back to an empty relays array rather than rejecting the whole bundle.
- 11 new regression tests added: 5 in `community.test.ts` (refounding quorum: trailing-slash dedupe, tolerant malformed-entry handling, all-malformed rejection, empty-string ack `from`, absent ack `from`) and 6 in `invite-bundle.test.ts` (non-string entries, empty/non-URL strings, `http://` scheme rejection, entirely-valid pass-through with no normalization, entirely-junk empty-array fallback, cap-then-filter ordering).
