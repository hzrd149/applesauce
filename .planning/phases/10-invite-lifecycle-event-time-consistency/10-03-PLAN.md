<objective>
Make all chunks of one Guestbook snapshot share a single instant — one `created_at` AND one `ms` tag — by threading a pre-computed `splitTime` pair from the caller (`buildSnapshotFactories`) into every chunk (TIME-02/D-08). This is the same single-read mechanism as TIME-01, applied one call-depth up because N chunks must share one instant by construction.

Purpose: Today each chunk's `created_at` is its own `blankEventTemplate` read and an explicitly-passed `ms` never reaches `created_at` at all — so a snapshot's chunks can straddle a second boundary.
Output: `buildSnapshotFactories`/`SnapshotFactory.create`/`includeSnapshotChunk` thread a `{ created_at, ms }` pair; snapshot test proves shared timestamp.
</objective>
