<objective>
Gate `refound()`'s adoption and compaction/snapshot publish on a per-wrap majority-confirmed root roll (D-09/D-11, ROTATE-09). Today every publish uses `.catch(() => {})` / `.catch(console.warn)`, discarding the `PublishResponse[]` (a resolved array never triggers `.catch`), and `adoptRefounding` runs unconditionally — so a Refounder can roll forward alone onto an epoch no one else can discover (M04).

Purpose: Define "confirmed publication" as strict majority (`> 50%`) of the configured relays per wrap, and abort (throw) if any gated wrap fails — matching D-01's abort-before-further-publish shape. The sequence is already correct (Pitfall 6); only the gating changes.
Output: A per-wrap majority gate over the awaited publish responses, before compaction/snapshot and adoption.
</objective>
