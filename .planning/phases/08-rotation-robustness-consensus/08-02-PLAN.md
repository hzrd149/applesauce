<objective>
Fix the first-arrival-wins consistency defect in `groupRotations` (D-02, ROTATE-10; and the identical `prevEpoch` gap, ROTATE-11). Today the first chunk parsed into a `(rotator, scope, newEpoch, prevCommit)` bucket fixes `set.chunkCount`, and `if (p.chunkCount === set.chunkCount)` silently drops any later chunk that disagrees — so a resumed rotation's stale generation can complete a set using only the first generation's chunks. `prevEpoch` has no cross-chunk validation at all.

Purpose: A rotation whose chunks disagree on `n` or `prevepoch` must be recognized as inconsistent (never `complete`, refetch), not silently completed on first arrival. Do NOT add `chunkCount` to the correlation key — that diverges from upstream and still lets a stale n-set complete.
Output: A `RekeyRotationSet.consistent` flag and a multiset agreement check in `groupRotations`.
</objective>
