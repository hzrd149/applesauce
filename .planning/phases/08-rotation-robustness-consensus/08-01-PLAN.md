<objective>
Land the down-only re-read spine and per-epoch anti-refork latch (D-04) — the single load-bearing mechanism that unblocks BOTH racing-rotation convergence (ROTATE-06/07) AND transient-decrypt passive retry (ROTATE-05). Today `syncEpochs` marks every non-tip epoch `"known"` and never re-invokes `readRekey` for it, and `rekeyHandled: Set<number>` blocks any second outcome — so a fork committed to (correctly or via a transient bug) can never be revisited by any code path.

Purpose: Convert "decide an epoch once, never revisit" into "revisit on every full walk, but accept only a strictly-lower sibling." This is the prerequisite plan; ROTATE-05/06/07's decision-logic plans (08-03) depend on it.
Output: A re-read walk (root + channel scope) plus a graded `Map<epoch, key>` latch that permits down-only re-convergence and forbids re-forking a settled epoch.
</objective>
