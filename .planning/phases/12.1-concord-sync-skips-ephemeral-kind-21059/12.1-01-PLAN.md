<objective>
Stop the Concord sync backfill path from requesting ephemeral kind 21059 (`EPHEMERAL_GIFT_WRAP_KIND`), which relays never retain under NIP-01, so a historical-fetch filter for it can only ever return nothing — wasting round-trips and producing a misleading "synced" signal. The fix is a single-site filter change plus a named constant that structurally guards against re-adding 21059 to the walk.

Purpose: Remove the dead ephemeral half of the backfill filter (D-01) while keeping the two live-subscription sites unchanged (D-02) — the asymmetry (backfill = retained kinds only; live = retained + ephemeral) is the intentional crux of the phase.
Output: An exported `BACKFILL_KINDS` constant in `gift-wrap.ts`, `syncAuthors` routed through it, and a two-directional, non-vacuous filter-spy regression test.
</objective>
